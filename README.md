# BackupFlow

BackupFlow - desktop-програма для безпечної двосторонньої синхронізації папки на комп'ютері з папкою на зовнішньому диску.

Поточна версія: `1.1.0`

## Що вміє програма

- створює окремі профілі синхронізації;
- дозволяє обирати папки через системний вибір папки;
- підтримує перемикання мови інтерфейсу: українська або English;
- використовує окремий font stack для українського інтерфейсу, щоб кирилиця виглядала природніше на macOS і Windows;
- аналізує обидві папки перед копіюванням;
- показує план синхронізації до виконання;
- після `Аналіз` використовує підготовлений план для `Синхронізувати`;
- копіює файли в обидва боки;
- використовує системні інструменти копіювання: `rsync` на macOS і `robocopy` на Windows;
- показує прогрес довгих операцій, поточний етап, файл, час і кількість дій;
- дозволяє перервати довгий аналіз або синхронізацію кнопкою `Стоп`;
- не видаляє файли назавжди під час видалення профілю;
- зберігає профілі, історію синхронізацій і metadata у SQLite;
- ігнорує типові системні файли: `.DS_Store`, `Thumbs.db`, `desktop.ini`, AppleDouble `._*`;
- нормалізує Unicode-імена файлів, щоб APFS/exFAT не створювали повторні копії для візуально однакових назв;
- для звичайного режиму порівнює файли за розміром і датою редагування;
- у режимі `Сувора перевірка` додатково перевіряє хеші малих не-медійних файлів, коли розмір однаковий, а дата відрізняється.

Головне правило безпеки: BackupFlow не має мовчки перезаписувати конфліктні файли.

## Швидкий Запуск

На macOS:

```text
start-macos.command
```

На Windows:

```text
start-windows.bat
```

Launcher створює локальне Python-середовище `.venv` у папці проєкту. Python-бібліотеки BackupFlow не встановлюються глобально в систему.

Перший запуск може тривати довше, бо встановлюються залежності frontend-частини і компілюється Tauri development-застосунок.

## Базовий Сценарій

За замовчуванням інтерфейс відкривається українською. Мову можна змінити в блоці `Мова інтерфейсу` в sidebar.

1. Натисни `Новий профіль`.
2. Введи назву профілю.
3. Обери `Папка на комп'ютері`.
4. Обери `Папка на зовнішньому диску`.
5. За потреби увімкни `Сувора перевірка`.
6. Натисни `Зберегти`.
7. Натисни `Аналіз`.
8. Перевір `План синхронізації`.
9. Натисни `Синхронізувати`.
10. Після завершення знову запусти `Аналіз`.

Якщо все синхронізовано, результат має бути `0 changes to sync`.

## Мова Інтерфейсу

BackupFlow підтримує дві мови UI:

- `Українська`;
- `English`.

Перемикач знаходиться в лівій панелі в блоці `Мова інтерфейсу`. Вибір зберігається локально в браузерному `localStorage` Tauri-вікна, тому після перезапуску програма відкриється останньою вибраною мовою.

Локалізується тільки desktop-інтерфейс: кнопки, заголовки, labels, helper text, статуси, progress panel і підтвердження видалення профілю.

Backend, CLI, JSON API і terminal logs залишаються технічними й не перекладаються.

## Аналіз І Синхронізація

`Аналіз` нічого не копіює. Він сканує обидві папки й готує план: що треба скопіювати, оновити, пропустити або обробити як конфлікт.

`Синхронізувати` виконує підготовлений план. Якщо перед цим був успішний `Аналіз`, повторний аналіз не запускається. Якщо підготовленого плану немає або він не підходить до поточного профілю, backend будує новий план.

## Сувора Перевірка

Звичайний режим швидкий: файл вважається однаковим, якщо збігаються шлях, розмір і дата редагування з невеликим допуском для різних файлових систем.

`Сувора перевірка` повільніша, але обережніша. Якщо розмір однаковий, а дата редагування відрізняється, BackupFlow може порахувати хеши вмісту і зрозуміти, чи файл реально змінився.

Щоб не гальмувати на великих файлах, великі файли, відео, аудіо, архіви й образи дисків навіть у strict-режимі перевіряються швидким способом: розмір плюс дата.

## Де Зберігаються Дані

Профілі та історія зберігаються в SQLite:

```text
~/.backupflow/backupflow.sqlite3
```

На Windows це зазвичай:

```text
C:\Users\<User>\.backupflow\backupflow.sqlite3
```

Локальні робочі файли launcher-ів:

```text
.venv
.backupflow
apps/desktop/node_modules
```

Вони не мають потрапляти в Git.

## Backend CLI

```bash
cd apps/backend
python3 -m backupflow init-db
python3 -m backupflow create-profile "My Projects" /path/to/local /path/to/external
python3 -m backupflow list-profiles
python3 -m backupflow analyze PROFILE_ID
python3 -m backupflow sync PROFILE_ID
python3 -m backupflow serve
```

Для ізольованих тестів можна задати окремий шлях до бази:

```bash
BACKUPFLOW_DB_PATH=/tmp/backupflow-dev.sqlite3 python3 -m backupflow serve
```

npm run build
