You are a senior software architect and lead developer.

Your task is to design and implement a cross-platform desktop application called BackupFlow.

The application must work on both Windows and macOS.

This is NOT just another file copy utility.

The goal is to create the fastest and safest synchronization tool for developers, photographers, videographers and anyone working with large projects.

=====================================================
MAIN IDEA
=====================================================

The application synchronizes selected folders between a computer and an external drive.

Typical workflow:

1. User connects an external SSD/HDD.
2. Launches BackupFlow.
3. Application detects the external drive.
4. User chooses synchronization profile.
5. Application analyzes differences.
6. Application shows synchronization plan.
7. User presses Synchronize.
8. Synchronization starts.

The application performs bidirectional synchronization.

If new files exist on the computer:
→ copy them to external drive.

If new files exist on external drive:
→ copy them to computer.

If files were modified:
→ synchronize newest version.

If both copies changed:
→ detect conflict.

Safety is more important than automation.

Never silently overwrite conflicting files.

=====================================================
FILE COMPARISON
=====================================================

DO NOT calculate hashes for every file.

Use this algorithm:

1. Compare path.
2. Compare size.
3. Compare modification time.

If all equal:
Skip.

If size differs:
Copy.

If modification time differs but size is equal:
Calculate xxHash64.

Only if hash differs:
Copy.

Never calculate hashes for every file.

=====================================================
VERY LARGE FILES
=====================================================

For video files larger than 5GB:

Use only:

- size
- modification time

Do NOT calculate hashes unless user enables "strict verification".

=====================================================
